# 操作系统 · 易错点总结类提示词

## OS-04 进程同步与死锁高频易错点

**适用场景**：做完进程同步（PV操作）和死锁章节的题目，想系统梳理易错点时使用。

**提示词**：

> 我正在复习考研408操作系统的进程同步与死锁部分，请列举5个考研中最容易犯错的易错点，要求：
> 1. 覆盖范围：信号量含义、PV操作顺序、死锁四条件、死锁预防/避免/检测的区别
> 2. 每个易错点格式：错误表述 + 正确说法 + 如何在做题时避免
> 3. 如果某个概念有多种说法，请以王道教材为准

**预期输出示例**：

```
易错点1：信号量初值的含义
❌ 错误：信号量初值代表"已有多少进程在等待"
✅ 正确：信号量初值代表"可用资源数量"（互斥信号量初值=1，同步信号量初值通常=0）
避免方法：做题时先写清楚每个信号量的含义再写P/V操作

易错点2：死锁预防 vs 死锁避免
❌ 错误：两者都是"防止死锁发生"，没区别
✅ 正确：预防=破坏四个必要条件之一（静态策略）；避免=动态分配资源时检测安全性（银行家算法）
避免方法：记住"预防静态、避免动态"
```

**使用注意**：

- AI 易错：死锁"检测"和"预防"、"避免"经常被混为一谈，注意区分三者
- 验证方法：对照教材中死锁处理策略的分类表格
- 追问方向：「请用生产者-消费者问题为例，讲解PV操作的正确写法」
