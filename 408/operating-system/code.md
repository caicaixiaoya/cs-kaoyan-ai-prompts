## OS-03 银行家算法模拟演示

**适用场景**：需要理解银行家算法的执行过程，或遇到银行家算法的计算题不知道如何下手时使用。

**提示词**：

> 请演示银行家算法的完整执行过程，要求：
> 1. 使用以下数据（3种资源A/B/C，4个进程P0~P3）：
>    - 可用资源 Available = [3, 3, 2]
>    - 最大需求 Max：P0[7,5,3] P1[3,2,2] P2[9,0,2] P3[2,2,2]
>    - 已分配 Allocation：P0[0,1,0] P1[2,0,0] P2[3,0,2] P3[2,1,1]
> 2. 先计算 Need 矩阵（Need = Max - Allocation）
> 3. 逐步执行安全性检查算法，找出一个安全序列
> 4. 每一步清楚展示 Work 向量的变化
> 5. 如果你在矩阵计算中不确定，请明确说明

**预期输出示例**：

```
Step 1：计算 Need 矩阵（Need = Max - Allocation）
P0: [7,5,3] - [0,1,0] = [7,4,3]
P1: [3,2,2] - [2,0,0] = [1,2,2]
P2: [9,0,2] - [3,0,2] = [6,0,0]
P3: [2,2,2] - [2,1,1] = [0,1,1]

Step 2：安全性检查
初始 Work = [3,3,2]，Finish = [F,F,F,F]

检查P1：Need[1]=[1,2,2] ≤ Work=[3,3,2] → 分配
  Work = [3,3,2] + [2,0,0] = [5,3,2]，Finish[1]=T
检查P3：Need[3]=[0,1,1] ≤ Work=[5,3,2] → 分配
  Work = [5,3,2] + [2,1,1] = [7,4,3]，Finish[3]=T
...
安全序列：P1 → P3 → P0 → P2
```

**使用注意**：

- AI 易错：银行家算法逐步执行时矩阵计算错误概率高，每步的 Work 向量要手动核查
- 验证方法：手动执行每一步，核对 Need 矩阵计算和 Work 向量更新
- 追问方向：「如果此时P1申请资源[1,0,2]，请判断是否可以分配」
