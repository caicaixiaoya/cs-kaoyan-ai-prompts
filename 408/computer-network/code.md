## CN-03 IP 数据报首部校验和计算演示

**适用场景**：需要理解 IP 首部校验和的计算方式，或想通过手算加深对协议细节理解时使用。

**提示词**：

> 请演示IP数据报首部校验和（Checksum）的计算过程，要求：
> 1. 解释校验和的计算规则：反码求和
> 2. 以以下简化的20字节IPv4首部为例，演示完整计算过程（十六进制）：
>    4500 0034 1c46 4000 4006 0000 ac10 0a63 ac10 0a0c
>    （注意校验和字段初始为0000）
> 3. 逐步展示每两个字节相加的过程，处理进位
> 4. 给出最终校验和值
> 5. 如果你对反码运算规则不确定，请明确说明

**预期输出示例**：

```
Step 1：将首部分成10个16位字：
4500, 0034, 1c46, 4000, 4006, 0000, ac10, 0a63, ac10, 0a0c

Step 2：逐一相加（十六进制）：
4500 + 0034 = 4534
4534 + 1c46 = 617a
617a + 4000 = a17a
a17a + 4006 = e180
e180 + 4006 → 有进位...
（最终和）取反码 = 校验和值
```

**使用注意**：

- AI 易错：十六进制加法中进位处理容易出错，特别是"将进位加回低16位"的步骤
- 验证方法：使用在线IP校验和计算器核对最终结果
- 追问方向：「接收方如何用校验和验证数据报是否出错？」
