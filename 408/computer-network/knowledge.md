## CN-01 TCP 三次握手与四次挥手详解

**适用场景**：需要搞清楚 TCP 连接建立/断开的完整过程，或对序列号变化规律有疑惑时使用。

**提示词**：

> 我正在复习考研408计算机网络，请详细讲解TCP三次握手和四次挥手，要求：
> 1. 三次握手：给出完整的报文交互过程，标注每步的 SYN/ACK 标志位和序列号/确认号的值（设客户端初始seq=x，服务器初始seq=y）
> 2. 四次挥手：同样标注标志位和序列号变化，说明为什么是"四次"而不是"三次"
> 3. 解释 TIME_WAIT 状态：为什么要等 2MSL，考研考什么
> 4. 如果你对序列号的具体变化规则不确定，请明确说明

**预期输出示例**：

```
三次握手：
① 客户端 → 服务器：SYN=1, seq=x（客户端进入SYN-SENT状态）
② 服务器 → 客户端：SYN=1, ACK=1, seq=y, ack=x+1（服务器进入SYN-RCVD状态）
③ 客户端 → 服务器：ACK=1, seq=x+1, ack=y+1（双方进入ESTABLISHED状态）

为什么是三次？
- 两次不够：服务器无法确认客户端能收到自己的消息
- 四次多余：第②步可以同时确认+发起，合并为一次
```

**使用注意**：

- AI 易错：序列号变化（seq和ack的值）在四次挥手中容易搞错，注意核查
- 验证方法：对照教材中TCP状态转换图，核查每步的状态变化
- 追问方向：「如果第三次握手丢失，会发生什么？TCP如何处理？」
