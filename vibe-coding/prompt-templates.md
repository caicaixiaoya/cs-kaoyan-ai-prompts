# Vibe Coding 提示词模板集

> 本文件收录 Vibe Coding 各阶段的提示词模板，可直接复制使用。
> 使用前将 `[方括号内容]` 替换为你的实际信息。

---

## VC-01 项目初始化 prompt

**适用场景**：项目开始阶段，让 AI 生成项目骨架时使用。

**提示词**：

> 我要用[技术栈，如Python/FastAPI/React等]做一个[项目简介，一句话]。
> 核心功能：
> - [功能1]
> - [功能2]
> - [功能3（如有）]
>
> 请帮我：
> 1. 生成项目目录结构（只包含这次需要的文件，不要过度设计）
> 2. 生成 requirements.txt 或 package.json（列出必要依赖及版本）
> 3. 生成项目入口文件（能成功运行，哪怕只打印 Hello World）
> 4. 给出本地运行的命令
>
> 注意：项目结构要简单，我是一个人在2周内完成。

**预期输出示例**：

```
目录结构：
project/
├── main.py          # 入口文件
├── requirements.txt # 依赖列表
├── config.py        # 配置文件
└── modules/
    ├── __init__.py
    └── core.py      # 核心逻辑

运行命令：
pip install -r requirements.txt
python main.py
```

**使用注意**：

- 收到目录结构后，先确认你理解每个文件的作用，再让 AI 生成代码
- AI 有时会生成过于复杂的结构，可以要求"进一步简化，去掉暂时用不到的"
- 追问方向：「请解释一下为什么要单独建config.py，不放在main.py里」

## VC-02 功能需求描述 prompt

**适用场景**：开发循环中，让 AI 实现一个具体功能时使用。

**提示词**：

> 我在做[项目名称]，当前已完成的功能：[已完成的功能列表，没有就写"无"]
>
> 现在需要实现：[用一句话描述这个功能]
>
> 具体要求：
> - 输入：[描述输入是什么]
> - 输出：[描述期望的输出是什么]
> - 边界情况：[描述需要处理的特殊情况，如"输入为空时..."，没有想到就写"暂不考虑"]
>
> 相关的现有代码：
> [粘贴相关的现有代码片段，让AI了解上下文]
>
> 请实现这个功能，并对关键代码行写注释。

**预期输出示例**：

```python
def process_user_input(text: str) -> dict:
    """处理用户输入，返回结构化结果"""
    if not text:  # 处理空输入边界情况
        return {"error": "输入不能为空"}

    # 核心处理逻辑
    result = {
        "original": text,
        "processed": text.strip().lower()  # 清理并标准化
    }
    return result
```

**使用注意**：

- 描述需求时要具体，"做一个用户系统"太模糊，"实现用户登录，输入用户名密码，验证后返回token"才够具体
- 收到代码后先读懂，不懂的地方用 VC-04 prompt 询问
- 追问方向：「请给这个函数加单元测试」

## VC-03 Bug 修复 prompt

**适用场景**：遇到 bug 或报错，需要 AI 帮助定位和修复时使用。

**提示词**：

> 我在运行[项目名称]时遇到了问题，请帮我排查。
>
> **期望行为：** [描述正常情况下应该发生什么]
>
> **实际情况：** [描述实际发生了什么，包括错误现象]
>
> **完整错误信息：**
> ```
> [粘贴完整的错误信息，包括堆栈跟踪]
> ```
>
> **出问题的代码：**
> ```
> [粘贴出问题的代码片段]
> ```
>
> **我已经尝试过：** [描述你已经尝试的解决方法，没有就写"无"]
>
> 请：
> 1. 解释这个错误的原因
> 2. 给出修复方案
> 3. 说明为什么这样修复可以解决问题

**预期输出示例**：

```
错误原因：
AttributeError 说明你在 None 对象上调用了方法。
第15行的 result.get("data") 中，result 在某些情况下为 None（当API请求失败时）。

修复方案：
在调用 .get() 前先检查 result 是否为 None：

if result is not None:
    data = result.get("data")
else:
    data = []  # 或其他默认值

原因：API请求失败时返回None，需要先验证响应再使用。
```

**使用注意**：

- 错误信息一定要完整粘贴，截图不如文本，方便 AI 分析
- 如果 AI 的修复方案没有解决问题，把"修复后仍然出现的新错误"继续粘贴给它
- 追问方向：「如何避免这类错误在其他地方也出现？」

## VC-04 代码解释 prompt

**适用场景**：看不懂某段代码，需要 AI 逐行解释时使用（面试前必用）。

**提示词**：

> 请帮我理解以下代码，要求：
> 1. 用一句话说明这段代码的整体作用
> 2. 逐行（或逐块）解释，用中文注释的方式，说明每行在做什么
> 3. 如果用到了特殊的语言特性或设计模式，请解释它是什么以及为什么这样用
> 4. 最后，如果我是面试官问"这段代码是怎么实现的"，你会怎么回答？给我一个参考答案
>
> 代码：
> ```
> [粘贴你看不懂的代码]
> ```

**预期输出示例**：

```
整体作用：这段代码实现了一个简单的LRU缓存，最多存储capacity个元素，超出后删除最久未使用的。

逐行解释：
from collections import OrderedDict  # 使用有序字典，保持插入/访问顺序

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity      # 缓存最大容量
        self.cache = OrderedDict()    # 用有序字典存储key-value，最近访问的在末尾

    def get(self, key):
        if key not in self.cache:     # 如果key不存在
            return -1                 # 返回-1（约定的"不存在"标志）
        self.cache.move_to_end(key)   # 存在则移到末尾（标记为"最近使用"）
        return self.cache[key]

面试参考答案：
"我用Python的OrderedDict实现了LRU缓存。OrderedDict会记住插入顺序，
通过move_to_end()把最近访问的元素移到末尾，这样字典开头始终是最久未使用的元素，
删除时直接popitem(last=False)即可..."
```

**使用注意**：

- 理解了解释后，试着合上解释自己重新表述一遍，这才是真正掌握
- 如果某行仍然不懂，单独追问那一行
- 追问方向：「请用一个更简单的例子帮我理解OrderedDict的工作原理」

## VC-05 重构优化 prompt

**适用场景**：项目基本完成后，想提高代码质量或性能时使用（时间充裕时可选）。

**提示词**：

> 请帮我审查和优化以下代码，要求：
> 1. 指出代码中存在的问题（可读性、性能、潜在bug）
> 2. 给出优化建议，按优先级排序（重要/次要/可选）
> 3. 对你认为最重要的问题，给出重构后的代码
> 4. 解释重构的理由，让我能在面试时解释"为什么这样写"
>
> 重要约束：
> - 只优化这段代码的功能，不要修改接口（输入输出保持不变）
> - 如果优化会影响可读性，请明确说明
>
> 代码：
> ```
> [粘贴需要优化的代码]
> ```

**预期输出示例**：

```
发现的问题：

[重要] 重复代码：第10-15行和第25-30行逻辑相同，建议抽取为函数
[重要] 裸异常捕获：except Exception 过于宽泛，应该捕获具体异常类型
[次要] 变量命名：tmp, res 不够清晰，建议改为 temp_result, final_result
[可选] 列表推导式可以替代第8行的for循环，更Pythonic

重构代码（针对重要问题）：

def _process_item(item):  # 抽取重复逻辑
    """处理单个item，返回处理结果"""
    ...

try:
    result = _process_item(data)
except ValueError as e:  # 捕获具体异常
    logger.error(f"数据格式错误: {e}")
```

**使用注意**：

- 优化后一定要重新测试所有功能，优化可能引入新的 bug
- 如果优化建议改动太大，可以要求"只做最小改动，保持代码结构不变"
- 追问方向：「请解释什么是Pythonic，给我5个常见的Python最佳实践」
